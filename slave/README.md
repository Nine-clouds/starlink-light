# STC8H 灯控从机

HC-12 无线接收 + 继电器控制 + 状态 EEPROM 保存。

## 硬件

- **MCU**: STC8H1K08, 内部 IRC 11.0592MHz
- **UART**: P3.6(RxD), P3.7(TxD) ← HC-12 无线模块
- **继电器**: P1.0（高电平 = 开灯，低电平 = 关灯）
- **LED**: P1.0（指示灯，开灯点亮）

## 功能

- 接收 HC-12 无线灯控命令（ON/OFF/TOGGLE/QUERY）
- 继电器驱动（高电平开灯）
- **状态 EEPROM 保存**: 延迟写入 + wear leveling 防磨损
- **上电恢复**: 断电后恢复上次状态
- **广播命令**: 执行不回复（避免 HC-12 半双工冲突）
- **看门狗**: ~8s 超时自动复位

## 部署

编译一次固件，用工具自动改地址批量烧录：

```bash
# GUI 版 (推荐)
python3 tools/slave_flasher_gui.py
# 或直接运行 tools/STC8H_Burn_Tool.exe

# 命令行版
python3 tools/slave_flasher.py slave/xxx.hex --flash COM3
```

**手动方式：**

1. 打开 `starlink-slave.uvproj`（Keil C51）
2. 修改 `DEVICE_ADDR`（第 17 行）：
   - 房间 1（晨曦）= `0x01`
   - 房间 2（微风）= `0x02`
   - 房间 3（星光）= `0x03`
   - 房间 4（月影）= `0x04`
   - 房间 5（云栖）= `0x05`
   - 房间 6（霁月）= `0x06`
3. 编译 → 烧录到 STC8H1K08
4. 重复 6 次（每次改地址）

## 协议

帧格式: `10 18 [地址] [命令] 18 10`

| 命令 | 值 | 说明 |
|------|-----|------|
| ON | 0x01 | 开灯 → P1.0 高电平 → 回复状态 |
| OFF | 0x02 | 关灯 → P1.0 低电平 → 回复状态 |
| QUERY | 0x03 | 查询 → 回复当前状态 |
| TOGGLE | 0x04 | 翻转 → 回复新状态 |

广播地址 `0xFF` 时执行命令但不回复。

## 回复帧

`10 18 [地址] [命令] [状态] 18 10`

状态: `0x01` = 开, `0x00` = 关
